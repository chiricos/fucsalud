# AGENTS.md

## Trust Config, Not Old Docs
- Root `README.md` is upstream Drupal boilerplate, not project-specific guidance.
- Some checked-in docs are stale: they still mention Drupal 9 / PHP 8.1 and `sites/default/files/sync`. Trust `composer.lock`, `.ddev/config.yaml`, and `sites/default/settings.php`.
- Actual stack: Drupal 10.6.8, PHP 8.2, MariaDB 10.11, nginx-fpm.
- Repo root is the Drupal docroot (`.ddev/config.yaml` has `docroot: ""`). Generic instructions that assume `web/` or `docroot/` are wrong here.

## Working Paths
- First-party code is not only under `modules/custom/`. There is additional repo-local module code directly under `modules/` (for example `fucs_payments/`, `biblioteca_form/`, `drawde_block/`, `drupalchat/`). Inspect before treating a module as Composer-managed contrib.
- Main theme is `themes/custom/fucs/` (base theme `classy`).

## Commands
- Prefer DDEV-wrapped commands: `ddev start`, `ddev drush cr`, `ddev drush updb`, `ddev drush uli`, `ddev drush cex`, `ddev drush cim`, `ddev composer <args>`.
- Focused PHP checks: `ddev exec vendor/bin/drupal-check <path>` and `ddev exec vendor/bin/rector process <path>`. `rector.php` is configured for Drupal 8/9/10 and also scans `.module`, `.theme`, `.install`, `.inc`, `.engine`.
- No repo-local frontend build/task runner or CI workflow is checked in. Theme CSS/JS is edited in place; `.eslintrc.json` exists but no npm scripts or theme `package.json` are present.

## Drupal-Specific Gotchas
- Actual config sync directory is `config/sync`; older docs pointing to `sites/default/files/sync` are stale.
- `sites/default/settings.ddev.php` is autogenerated by DDEV; do not hand-edit it.
- `sites/default/settings.php` hard-codes `$settings['file_public_base_url']` to `https://www.fucsalud.edu.co/sites/default/files`. Local file URL bugs may come from this config, not from Twig/theme changes.
- `.ddev/config.override.yaml` runs a `post-start` hook that writes global Composer config for a private GitLab registry using `$GITLAB_TOKEN`; startup noise here is not a Drupal error.

## Rendering Architecture
- `themes/custom/fucs/fucs.info.yml` defines many named regions; block placement is part of page structure, not just Twig.
- `themes/custom/fucs/fucs.theme` injects blocks from selected regions into full node renders via `fucs_add_regions_to_node()`. When a node page looks wrong, check region blocks as well as the Twig template.
- Several custom modules render via their own `hook_theme()` and module-local `templates/` directories. Do not assume every Twig file lives in the theme.
- `modules/custom/fucs_form/` owns the admission/induction form flow and PDF certificate generation via `dompdf`; the certificate markup is `modules/custom/fucs_form/templates/certificate.html` (plain HTML, not Twig).
- Some logic targets hard-coded block plugin IDs/UUIDs (for example `fucs_form_block_view_alter()` in `modules/custom/fucs_form/fucs_form.module`). Grep for `block_content:` before changing block placement or weights.
