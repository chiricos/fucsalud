## Drupal-Specific Gotchas
- Config sync directory: `config/sync`. Older docs pointing to `sites/default/files/sync` are wrong.
- `sites/default/settings.ddev.php` is autogenerated by DDEV; do not hand-edit it.
- `settings.local.php` include is **commented out** in `settings.php` — no local override loads automatically by default.
- `$settings['file_public_base_url']` is **not set** in `settings.php` (only appears as a commented-out upstream example). If local file URLs are broken, the cause is elsewhere.
- `composer audit` will always exit clean — 15 security advisories are intentionally suppressed in `composer.json`'s `config.audit.ignore`.
- Notable non-obvious Composer dependencies: `dompdf/dompdf` (PDF certs), `google/apiclient` (OAuth), `nguyenanhung/nusoap` (legacy SOAP client), `phpmailer/phpmailer`.

## Rendering Architecture
- `themes/custom/fucs/fucs.info.yml` defines many named regions; block placement is part of page structure, not just Twig.
- `themes/custom/fucs/fucs.theme` injects blocks from selected regions into full node renders via `fucs_add_regions_to_node()`. When a node page looks wrong, check region blocks as well as the Twig template.
- Several custom modules render via their own `hook_theme()` and module-local `templates/` directories. Do not assume every Twig file lives in the theme.
- `modules/custom/fucs_form/` owns the admission/induction form flow and PDF certificate generation via `dompdf`; the certificate markup is `modules/custom/fucs_form/templates/certificate.html` (plain HTML, not Twig).
- Some logic targets hard-coded block plugin IDs/UUIDs (e.g. `fucs_form_block_view_alter()` in `modules/custom/fucs_form/fucs_form.module`). Grep for `block_content:` before changing block placement or weights.

## Xdebug
- `ddev drush-xdebug` is **not** in `.ddev/commands/web/` by default. Copy it from the skill before use:
  ```bash
  cp .github/skills/drupal-xdebug/scripts/drush-xdebug .ddev/commands/web/drush-xdebug
  chmod +x .ddev/commands/web/drush-xdebug
  ```
- Port **9003** → PHPStorm (web/HTTP). Port **9010** → OpenCode xdebug-mcp (CLI/Drush). Both coexist.
- Breakpoints require **container paths**: `/var/www/html/...`, not host paths.
- `xdebug_enabled: false` in `.ddev/config.yaml` is intentional; enable on demand with `ddev xdebug on` / `ddev xdebug off`.
- Activating xdebug MCP tools in the agent requires `"mcp": { "xdebug": { "enabled": true } }` in `opencode.json` (user-level config, not in this repo).
